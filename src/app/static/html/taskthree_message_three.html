Since the server is taking our input and turning it into pages, we should test this for any possible exploits or venues of attack. Typically this is done by submitting various strings that could be malicious and seeing what happens.

Let's start with something simple - javascript! If we can submit javascript code that executes, it might lead us to getting the flag itself. A very primitive way of testing that would be to submit the following:

<script>alert(1)</script>

That code will execute the alert(1) command as soon as it is loaded, which will create a pop-up alert that contains the message "1". This means that we can submit that as the content of the essay, navigate to where the essay is displayed, and if we receive a javascript alert, then that means our submitted javascript will execute to anyone who visits it!
