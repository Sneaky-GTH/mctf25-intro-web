Magnificent! We just exploited an exposed endpoint! However, take a look at the source code again - did you notice there's two blocks of code, not just one? Let's take a look at the second one:

document.getElementById('explore-btn').addEventListener('click', () => {
   fetch("/taskone/hidden-datav-post", {
      method: 'POST',
   )
   .then(response => response.json())
   .then(data => {
      console.log(data);
   })
   .catch(error => console.error('Error:', error));
});

This does pretty much the exact same thing as before, but with a major difference - notice the method: "POST". When we previously visited /taskone/hidden-data, we did that using the GET, which, in simple terms (in reality, it's a bit more complicated...), is like visiting that website directly via the browser.

POST, on the other hand, is more like sending only specific data back and forth without visiting the whole website. To see what I mean, try visitting /taskone/hidden-data-post!
