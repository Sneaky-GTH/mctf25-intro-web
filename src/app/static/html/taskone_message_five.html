Well... that didn't do much. Remember what we just discussed? Let's try checking the source code for what exactly this button does.

I added it here for your convenience, though I recommend finding it in the original source code!

<script>
document.getElementById('explore-btn').addEventListener('click', () => {
fetch("/taskone/hidden-data")
.then(r => r.json())
});

Let's take it bit by bit:

  • The <script> means that we're no longer working with HTML (think of it as no longer working with text) but with JavaScript (a programming language)
  • It finds an element which has the id "explore-btn" (try finding it in the source code!) and executes code whenever it is clicked.
  • When clicked, it executes fetch("/taskone/hidden-data") and returns data.

fetch() is a javascript function that basically says "get the contents of this website". If you put in, for instance, fetch("google.com"), it would try and "fetch" a resource from google.com, though google probably wouldn't return anything. In this case, it seems to be fetching "/taskone/hidden-data" - notice how it doesn't include a hostname. This means it'll default to the current hostname. The hostname can be seen at the top of the browser - in this case, that would be the IP address (something like "172.12.0.4", though it could also be a domain - think "google.com").

So, whenever we click the button, our browser effectively visits http://172.12.0.2/taskone/hidden-data (the IP address might be different in your case), receives data from it, and.... doesn't do anything. Now - what if we were to visit the website ourselves?
