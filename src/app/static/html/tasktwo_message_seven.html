As before, take a look at the top of the requests - there should be a request with the POST method that is sending a request to buy, though depending on your chosen product it may differ a bit.

Did you notice that it no longer includes data in the "file" section (the "?product=hat&price=5" part)? That's because using POST, the data is transmitted a bit differently!

Click on the request, look over to the newly-opened box on your right and navigate to the Request section. This is where the data that you send is shown - instead of being a part of the "address" or "link", it's instead part of the request itself!

However, we can still use the exact same exploit we did previously, just with a different method! Try right clicking on the request, head to the Copy Value option and select Copy as cURL. Next, open a terminal of your choice and paste in the content (typically Ctrl+Shift+V if you're using a Linux terminal). It should look something like this:

curl 'http://172.12.0.5/tasktwopost/buy'
-X POST
-H 'User-Agent: Mozilla/5.0 (X11; Linux x86_64; rv:145.0) Gecko/20100101 Firefox/145.0'
A bunch of lines starting with -H that are not important to us right now...
-H 'Cache-Control: no-cache'
--data-raw 'product=hat&price=5'

See that last line? That's the data that's being sent! Simply change the price and use the same exploit we used previously. After entering the command, our terminal will then send a POST request to the given address and data - after sending it, all that is left is to refresh the page.
